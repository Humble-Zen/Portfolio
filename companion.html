Companion AI

A modular survivor companion for a third-person zombie game. Follow behaviour, hiding, autonomous fleeing, bottle throwing, and scavenging all driven by a clean state machine and a decoupled event layer. Four scripts, no tight coupling between them.

States in Action
Follow state. The companion trails behind the player along their movement direction, staying out of the shooting line.
Hiding state. On CommandHide() she sprints to the nearest hiding spot and crouches. Immune to zombie detection while hidden.
CommandClear(). She exits the hiding spot and returns to Follow. Player confirms the area is safe.
Scavenging state. On CommandScavenge() she walks to the nearest LootSpot, waits the scavenge timer, then returns with supplies.
Fleeing state. After taking a hit with enough zombies nearby, she sprints to a sampled NavMesh point away from the threat cluster.
Architecture

CompanionAI is the single source of truth. All other scripts either call its public command methods or subscribe to its static events. Neither holds a direct reference to the other.

  • CompanionAI.cs core state machine, NavMesh movement, perception, all commands
  • CompanionInputHandler.cs input bridge, maps keybinds to CompanionAI commands, subscribes to events for HUD feedback
  • BottleProjectile.cs physics projectile, parabolic arc, target leading, collision, break effects
  • LootSpot.cs scene loot node, weighted random table, pick tracking, visual props, depletion events
Design Patterns Used
  • Singleton CompanionAI.Instance, one companion per scene, globally accessible without scene-coupling
  • State Machine CompanionState enum with EnterState() transitions and per-state Tick_ methods, easy to extend without touching existing states
  • Observer (Event-Driven) five static C# Actions on CompanionAI; CompanionInputHandler and HUD scripts subscribe independently, no references needed
  • Velocity-Based Follow Anchor anchor placed behind the player along their smoothed velocity direction (not companion-relative offset); fixes the common 'locked in front' bug
  • Action Lock Coroutine a shared bool (_actionLock) blocks locomotion animator writes during one-shot clips (throw, react), preventing the sliding-in-flinch-pose bug
  • ScriptableObject-style Loot LootSpot owns its own weighted loot table per instance, falling back to CompanionAI's global table; designers tune each spot in the Inspector
Public Command API
  • CommandFollow() force return to Follow from any state
  • CommandHide() move to nearest hiding spot
  • CommandClear() exit Hiding or Fleeing, return to Follow
  • CommandThrowBottle() throw at nearest zombie, silent no-op if invalid
  • CommandScavenge() move to nearest loot spot
  • HealCompanion(float amount) add HP, clamped to maxHealth
  • GiveBottles(int count) add bottles, clamped to maxBottleCount
Events Fired
  • OnZombieSpotted Action<Transform>, fires when first zombie enters perception radius
  • OnCompanionDied Action, fires when health reaches 0
  • OnItemScavenged Action<string>, fires on loot pick completion, passes item name
  • OnBottleThrown Action<int>, fires on launch, passes remaining count
  • OnBottlesEmpty Action, fires when last bottle is used
Seven States

Transitions are driven by player commands, proximity checks, and damage events never polled externally.

StateEntryBehaviourExit
Follow Default / CommandFollow() Trails player via velocity anchor. Idles and wanders locally when player is stationary. Any command, or damage while zombies are nearby
MovingToHide CommandHide() Sprints to nearest hiding spot at hideRunSpeed. Arrives within hideArrivalRadius → Hiding
Hiding Arrival at hide spot Crouches, immune to zombie detection. CommandClear() or CommandFollow()
Fleeing Hit by zombie + fleeZombieThreshold nearby Sprints to a sampled NavMesh point away from threat cluster, then crouches. CommandClear()
MovingToScavenge CommandScavenge() Walks to nearest LootSpot. Aborts if zombie enters alertScanRadius. Arrives at spot → Scavenging; zombie spotted → Follow
Scavenging Arrival at LootSpot Waits scavengeDuration seconds, then calls LootSpot.Loot(). Timer expires → Follow
Dead Health reaches 0 Terminal. No transitions out. OnCompanionDied fires. None
Follow Anchor System

The anchor is the world-space position the companion navigates toward in the Follow state. Each frame, player position delta is blended into _smoothVelocityDir using Vector3.Slerp with anchorBlendSpeed. The anchor is placed behind the player along the smoothed velocity direction with a perpendicular lateral offset.

During a sprint (speed > sprintThreshold), lateral offset fades to zero and trailing distance grows, keeping the companion directly behind and out of the shooting line. A hard push-back kicks in if the companion gets inside walkCatchUpRadius.

behindDir = -_smoothVelocityDir moveRight = Cross(Vector3.up, _smoothVelocityDir) // travel-relative, not camera-relative anchor = player.pos + behindDir * activeDistance + moveRight * activeLateral
Parabolic Bottle Arc

CalculateArcVelocity() derives the exact initial velocity to hit the target in a single parabolic arc. arcHeight is measured above whichever endpoint is higher, so the arc reliably clears geometry regardless of height difference.

peakY = Max(start.y, end.y) + arcHeight timeToPeak = Sqrt(2 * riseH / gravity) timeToLand = Sqrt(2 * fallH / gravity) velXZ = dispXZ / totalTime velY = gravity * timeToPeak

Rigidbody drag is forced to 0 before launch any drag causes the trajectory to deviate and the bottle falls short. Angular drag (0.05) is left on for a natural-looking spin.

Target Leading

The target position is offset by the zombie's NavMeshAgent velocity × targetLeadTime (0.25s default), with a Rigidbody fallback. This keeps throws accurate against moving targets.

Bottle Throw Safety Checks

CommandThrowBottle() is a silent no-op if any of these are true: companion is dead or hiding, no bottles remain, cooldown has not expired, or no zombie is within bottleTargetRadius. If valid, it plays the ThrowBottle animator trigger, locks locomotion via the action-lock coroutine, spawns BottleProjectile, and decrements _bottles.

Collision and Startle

OnCollisionEnter and OnTriggerEnter both call owner.OnBottleHitZombie(), which deals damage and calls ZombieAI.NotifyStartle() treating the hit as a mid-priority distraction alert. Break particles and audio detach from the projectile before destruction so they finish playing after the GameObject is gone.

Loot System
  • Weighted random table each LootSpot carries LootEntry (itemName, amount, weight). PickWeightedRandom() rolls proportionally; a weight-5 item is five times as likely as a weight-1 entry.
  • Scavenge flow CommandScavenge() finds nearest non-depleted LootSpot via OverlapSphere. On arrival, scavengeTimer counts down. On expiry, LootSpot.Loot() is called, item name passed to OnItemScavenged, then returns to Follow.
  • Abort on threat any zombie entering alertScanRadius during transit cancels the scavenge and returns to Follow.
  • Scene gizmos LootSpots draw a wire cube and upward ray in Scene view, label shows remaining picks (e.g. '2/3 picks') or 'DEPLETED'. Useful for level design without selecting each spot individually.
Perception and Alert Scan

CR_ZombieAlertScan() is a coroutine running on alertScanInterval (0.8s default). It does an OverlapSphere at alertScanRadius, filters by line-of-sight (Linecast against obstacleLayer from eye height), and populates _knownZombies, a HashSet<Transform> that pathfinder queries draw from. Dead zombies are removed each scan pass. The companion does not auto-hide on detection perception only drives voice barks and informs the flee threshold check.

Animator Contract

DriveAnimator() is called at the end of every Update. It sets locomotion bools based on agent speed, but only if _actionLock is false preventing overwrites mid-throw or mid-react. Critical setup: the 'React' Animator state must have its Tag set to 'React', and the 'Throw' state must have its Tag set to 'ThrowBottle'. The action-lock coroutine polls the current state tag to know when the clip has exited. Without these tags, the companion freezes after the first hit or throw.
