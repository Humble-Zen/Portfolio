Unit Navigation A* in Action
The blue player unit uses A* pathfinding to navigate around red obstacle spheres on the grid. "Player initialized with PathFinder" confirms the system is live. The black capsule is the enemy AI unit.
Custom Editor Tool Opening the Tool
The custom ObstacleEditor is accessed directly from Unity's menu bar, no inspector setup needed. Designers open it like any native Unity window. The isometric grid is visible in the background with existing obstacles already placed.
Custom Editor Tool Placing Obstacles
The ObstacleEditor renders the grid as a checkbox matrix. Each checkbox maps to one tile. Click to block, click again to unblock. Changes persist via EditorUtility.SetDirty(), no code, no prefab editing, changes survive play sessions. The runtime reads the same data to spawn obstacle prefabs.
Turn Structure
  • Player clicks a destination then PathFinder routes around obstacles and occupied tiles, unit animates tile by tile, AI responds immediately after.
  • GameLock static flag blocks all input during movement, prevents mid animation input and keeps turns feeling deliberate.
  • No random timers. AI reacts the moment the player coroutine completes.
A* From Scratch

PathFinder is a plain C# class with no MonoBehaviour dependency. It can be tested, ported, or called from anywhere.

  • FindPath() resets all node costs before each search stale g/h/f values from a previous call won't corrupt the result.
  • Open set is a List<Node> sorted by f cost. Closed set is a HashSet<Node> for O(1) membership checks.
  • Path is reconstructed by walking parent references from goal back to start, then reversing.
// Manhattan heuristic grid movement, no diagonals h = Mathf.Abs(a.x - b.x) + Mathf.Abs(a.y - b.y);
Editor Tool

Level designers can block tiles without touching code.

  • ObstacleData ScriptableObject stores a flat bool[] array. x,y → index via y * width + x. Fully serializable, no jagged arrays.
  • ObstacleEditor (EditorWindow) renders the grid as checkboxes. Click to block, click again to unblock. Saved via EditorUtility.SetDirty() persists across play sessions and into version control.
  • At runtime, ObstacleManager reads the same ScriptableObject and spawns prefabs at blocked positions. No data duplication.
Tile Occupancy
  • Pre-reservation GridOccupant.Occupy() is called on the destination before the unit moves into it. Two units can't target the same tile.
  • HashSet<Vector2Int> for O(1) checks. PathFinder queries it during neighbour generation so paths route around occupied tiles automatically.
Enemy AI

EnemyAI never polls in Update(). It subscribes to PlayerControl.OnPlayerMovedEvent in OnEnable and unsubscribes in OnDisable responds only when the player actually moves.

  • IAI interface with a single TakeTurn() method. Swap AI behaviour without touching the event wiring.
